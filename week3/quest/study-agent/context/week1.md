# 1주차 — 개발 환경 설치 및 AI 에이전트 실습

## 학습 목표
AI 코딩 시대의 기본 도구들을 설치하고 사용법을 익힌다. AI 에이전트와 대화하며 실제 결과물(레시피, K-Drama)을 만들어본다.

## 설치한 도구

| 도구 | 역할 | 비고 |
|------|------|------|
| **Claude Code** | 터미널 기반 AI 코딩 에이전트 CLI | Anthropic 공식 |
| **Cursor** | AI 기반 코드 에디터 | VS Code 파생 |
| **Git CLI** | 소스코드 버전 관리 | v2.53.0 |
| **Vercel CLI** | 웹 프로젝트 배포 플랫폼 | v50.17.1 |

## 핵심 개념

### 채팅 vs 에디터 vs 에이전트
- **채팅**: 대화형 AI 인터페이스 (Claude.ai, ChatGPT)
- **에디터**: AI 기능이 내장된 코드 편집 도구 (Cursor)
- **에이전트**: 자율적으로 파일을 읽고 쓰며 작업을 수행하는 AI (Claude Code)

### Text Context
- AI에게 제공하는 "맥락(context)"이 결과 품질을 결정한다.
- 파일, 코드, 지시사항 등 모든 것이 **텍스트 기반**으로 전달된다.
- "파일 시스템도 모두 텍스트" — HTML, CSS, JS, JSON, MD 등 모든 파일은 결국 텍스트. AI가 파일을 읽고 쓸 수 있는 이유.

### Git & GitHub
- 기본 흐름: `git add` → `git commit` → `git push`
- GitHub을 통한 원격 저장소 관리
- harbor.school 2기 주말반은 `harbor-school` 단일 레포에 `weekN/` 폴더 구조로 정리
  - `weekN/practice/` — 수업/연습 코드
  - `weekN/quest/` — 수업 부여 퀘스트
  - `weekN/goblin/` — 자유 소규모 프로젝트 (한글 폴더명)

## 퀘스트 1 — 나만의 레시피 북 만들기 🍳

### 컨셉
**삼국시대 음식 복원 대결 — 고구려 vs 백제 vs 신라**

잊혀진 삼국시대 음식을 현대 대표 지역의 DNA로 복원한 레시피 3종.

| 나라 | 현대 지역 | 레시피 |
|------|-----------|--------|
| 🔴 고구려 | 개성 | 된장맥적 (불고기의 원조) |
| 🟢 백제 | 전주 | 민물어죽 (보양 쌀죽) |
| 🟣 신라 | 안동/영주 | 발효 보리반상 (간고등어+나물) |

### 에이전트 구성
- **역사학자** — 삼국시대 음식 문화 고증
- **맛전문가** — 현대 지역 특색과 삼국 DNA 매칭
- **시각전문가** — 이미지·비주얼 구성
- **요리사** — 탕·구이·반상 각 카테고리 전문

### 학습 포인트
- 에이전트 역할 분담(페르소나)으로 다각도 조사
- 단일 프롬프트가 아닌 **여러 에이전트와의 5회+ 대화**로 결과물 완성
- 역사 고증 → 현대 지역 매칭 → 레시피 완성의 파이프라인

## 퀘스트 2 — 내 일기 K-Drama로 바꾸기 🎬

### 컨셉
**「같은 길, 다른 온도」 — 단편드라마 (반전 엔딩)**

평범한 일기 속 "반복과 변화"를 월~금 5일 구조의 단편드라마로 변환.

### 에이전트 활용
| 단계 | 대화 | 결과 |
|------|------|------|
| 1차 | 원본 일기 → 단편드라마 변환 요청 | K-Drama 포맷 초안 |
| 2차 | "반전 엔딩을 추가해달라" 요청 | 반전 엔딩이 포함된 완성본 |

### 학습 포인트
- AI 에이전트와의 **연속 대화(멀티턴)** 개념
- "한 번에 완벽한 결과를 바라지 말고, 이어서 다듬어라"
- 2회 대화만으로 일기가 드라마로 변신
